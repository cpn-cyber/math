# 问题1智能评估系统升级说明

## 1. 定位调整

问题1不再表述为多个常规模型的简单串联，而表述为“多维文本特征驱动的可审计智能评估系统”。系统由文本解析、结构识别、指标抽取、组合赋权、基础排序、规则化成对比较复核、等级划分、稳健性审计和证据解释九个环节构成。需要注意，这里的Agent是按评审职责划分的证据模块，不是独立调用大模型的黑箱评审器。

## 2. 多智能体评审框架

| Agent | 职责 | 对应指标或输出 |
|---|---|---|
| 结构规范 Agent | 检查摘要、章节、参考文献、附录和图表编号 | A1结构规范性、参考文献规范率、附录代码存在性 |
| 逻辑一致 Agent | 检查问题重述、假设、结论之间的逻辑链 | A2问题理解与逻辑严密性、结果-结论一致性 |
| 数学建模 Agent | 检查模型数量、公式密度、变量定义和约束完整性 | A3方法合理性与数学建模质量 |
| 结果验证 Agent | 检查结果完整性、图表解释、灵敏度和误差分析 | A4结果分析与验证 |
| 综合仲裁 Agent | 融合各Agent证据，输出最终分数、等级和风险提示 | AHP-熵权、TOPSIS、规则化BT复核、KMeans/Jenks分级 |

## 3. 成对比较校准的表述修正

当前 `winner` 并非真实线下专家人工填写，而是由基于五维rubric的规则化 surrogate reviewer 生成。因此论文中应表述为“规则化成对比较复核”，不要写成“人工判断”。Step 11 已读取 `external_anchor_blind_review_filled.xlsx`，检测到12对有效边界论文复核，其中9对与规则化winner一致，一致率为75.00%。该结果可作为外部锚补强，但覆盖范围仅限边界论文对，不能解释为全部30篇论文的外部真值。

## 4.1 Claude评委意见专项修订

- BT独立性审计：BT潜在排序与TOPSIS基础排序Spearman为0.189321，Kendall tau为0.135632，成对方向独立性指数为0.768116；该结果只能说明规则化BT与TOPSIS不完全重复，不能替代外部有效性。
- 外部锚复核：12对边界论文全部有效填写，与规则化winner一致9对，一致率为75.00%；A02、A06、A10为不一致样本对，应作为边界复核差异案例说明。
- AHP alpha敏感性：\(\alpha\in\{0.4,0.5,0.6,0.7\}\) 时，与最终排名的Spearman最小为0.968854，说明最终排序对组合系数变化较稳健。
- 稀疏二值指标审计：同时剔除I16灵敏度分析存在性和I04附录代码存在性后，Spearman降至0.727697，说明这两个可复现/验证信号对排序影响较大，正文需要公开解释其权重意义。
- 软指标消融：同时剔除I13、I20、I21后，Spearman仍为0.995551，说明软指标没有单独支配最终排序。
- 绝对护栏审计：18.txt最终等级为良好，但核心章节完整率为0.416667，触发“相对良好但结构完整性需复核”的风险提示。

## 6. 论文表达建议

- AHP部分不要过度强调CR=0，而应说明其为基于预设重要性向量构造的一致偏好矩阵。
- KMeans/Jenks部分不要写成两个完全独立模型共同证明，而写成“KMeans主分级，Jenks自然断点一致性校验”。
- BT部分明确规则化rubric来源，并将其定位为边界复核证据，避免“人工判断”不实表述。
- 对02.txt等边界论文给出等级置信度和风险提示，体现真实评审系统的不确定性意识。
- 对25.pdf说明OCR质量代理和风险记录，强调扫描件已进入可追溯质量控制流程。
- 对18.txt说明其相对排名为良好但结构完整性不足，属于需复核风险样本。
- 外部锚复核已完成，可在正文写入12对有效复核和75.00%一致率；但专家AHP模板尚未真正进入权重模型，不能写成已完成专家AHP重新赋权。
- 当前 `external_anchor_blind_review_filled.xlsx` 中 reviewer_id 已统一为 `R1`。
